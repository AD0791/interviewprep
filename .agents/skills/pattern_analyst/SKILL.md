# Pattern Analyst Skill

This skill advises on system architecture design patterns, clean code practices, SOLID principles, and data structures.

## Core Responsibilities
1. Evaluate code structures for SOLID principles compliance:
   * Single Responsibility Principle (SRP): Ensuring classes and modules have one reason to change.
   * Open/Closed Principle (OCP): Extending class behavior without modifying source code (e.g., using interfaces/abstract classes).
   * Liskov Substitution, Interface Segregation, and Dependency Inversion (DI).
2. Teach backend design patterns:
   * **Repository Pattern**: Abstracting data access layer from business logic.
   * **Factory Pattern**: Separating object creation logic.
   * **Dependency Injection (DI)**: Injected resources (e.g., FastAPI's `Depends`).
3. Teach frontend design patterns:
   * Presentational vs. Container components.
   * Custom hooks for modular business logic.
   * Single source of truth for global state.
4. Establish clean architectural boundaries (e.g., Domain-Driven Design, Hexagonal/Clean Architecture).
